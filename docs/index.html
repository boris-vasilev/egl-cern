get /data_links
Endpoint for searching through the data transfers that have completed until the time this request is made (dataLinksGet)

Query parameters

source (optional)
Query Parameter — The site name from where transfers originate from
destination (optional)
Query Parameter — The site name from where transfers terminate
vo (optional)
Query Parameter — The VO associated with the transfers. Example: 1) alice 2) atlas
technology (optional)
Query Parameter — The data transfer protocol used. Valus is one of xrootd or fts.
starts_between (optional)
Query Parameter — The time range in which the transfers begin. Format: {begining of time range}, {end of time range}. The values can NOT be influx db query parameters like now()-6h, now(). The values can ONLY be integers representing UNIX timestamps (number of seconds elapsed from epoch). Examples: 1) /data_links?starts_between=1560851060,1563443060 2) data_links?ends_between=now()-3h,now()&amp;limit=5000&amp;starts_between=1564358400,1564494652
ends_between (required)
Query Parameter — One of starts_between and ends_between is REQUIRED. The time range in which the transfers finish. Format: {begining of time range}, {end of time range}. Value can be integer timestamps or influx db query parameters like now()-6h, now(). Examples: 1) /data_links?ends_between=1560851060,1563443060 2) /data_links?ends_between=now()-1h, now()
minimum_transferred_volume (optional)
Query Parameter — The minimum threshold for transferred volume. Unit is yet to be revtrieved from the monitoring team.
maximum_transferred_volume (optional)
Query Parameter — The maximum threshold for transferred volume. Unit is yet to be revtrieved from the monitoring team.
limit (optional)
Query Parameter — Limit the number of results. At present, we query two transfer databases. The limit is set on the query made to each database. Therefore, if the limit is set to x, the total number of results retrieved by the server are 2x. After validation checks, the maximum results returned by the server are 2x.

get /sites
Gives a list of all the Sites in WLCG (sitesGet)
Parses the CRIC database of WLCG sites and federations, the Google Earth KML and combines the information into an Array of Site Objects.

Query parameters

name (optional)
Query Parameter — The name of the site
tier (optional)
Query Parameter — Filter results based on tier. Value can be one of 0,1,2,3
supported_vos (optional)
Query Parameter — Filter results based on supported_vos. Examples: 1) /sites?supported_vos=alice 2) /sites?supported_vos=alice,atlas,cms
country (optional)
Query Parameter — Filter results based on country Example /sites?country=Germany
country_code (optional)
Query Parameter — Filter results based on country code. Example /sites?country_code=DE
federation (optional)
Query Parameter — Filter results based on federation name
schema_version (optional)
Query Parameter — Request a particular schema version for the results
